# 03_PHASES — 콘텐츠 접근 보안 단계별 계획

## Phase 0 — 즉시 강화 (정적, 인프라 변경 0)
지금 당장 위험 낮추기 — Vercel 이전 전에라도.
- [ ] 검사결과 페이지 `no-store` + `X-Robots-Tag: noindex` + referrer-none 메타/헤더 일괄 점검.
- [ ] 해시 슬러그 엔트로피 확인(현 10자 hex 유지·충분성 점검), QR에 URL 텍스트 노출 없는지.
- [ ] 공유 가이드: 검사결과 링크는 "당일 열람" 안내(운영 룰).
**완료**: PII 자료의 검색·캐시 노출 0, 운영 룰 문서화.

## Phase 1 — 보호 티어 호스팅 + 만료 서명링크 (MVP, 핵심)
- [ ] 보호 콘텐츠(lab-reports)를 **Vercel**에 배치(별도 프로젝트 or 경로). T0 공개자료는 GitHub Pages 유지.
- [ ] 엣지 미들웨어: `protected_kinds` 경로에 **HMAC 서명 토큰 + 만료** 검증(fail-closed). 만료/무효 안내 페이지.
- [ ] 발급 도구(CLI/스킬 명령): 자료+TTL → 서명 링크/QR 생성. 기본 TTL=config.
**완료**: 검사결과 링크가 만료 후 차단, 원장이 1스텝으로 발급.

## Phase 2 — 이름+환자번호 게이트 (2차 인증)
- [ ] 빌드 시 자료에 `chart_no_hash`·`name_norm` 임베드(평문 금지).
- [ ] 엣지에서 이름·환자번호 폼 검증 + 시도 rate-limit + 짧은 세션쿠키.
- [ ] config `require_name_chartno`로 ON/OFF(자료별/전역).
**완료**: 만료링크 + 이름·번호 결합 인증 동작.

## Phase 3 — 검진결과 확장 + 무력화/운영
- [ ] `protected_kinds`에 `checkup-results` 추가만으로 검진결과 페이지가 같은 보호 적용되는지 검증(확장성 테스트).
- [ ] SigningKey 회전 절차(유출 대응=전체 토큰 무효) + 자료별 revoke.
- [ ] (선택) 최소 AccessLog(시각·성공/실패, PII 평문 0) + 이상 접근 알림.
**완료**: 새 종류 1줄 추가로 보호, 유출 시 일괄 무력화 가능.

## 의존성
P0(즉시) ∥ P1(키스톤) → P2 → P3. 검진결과 페이지 신설은 별개 작업이나 P3의 config 슬롯이 미리 받쳐줌.
